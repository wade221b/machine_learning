# Cheat Sheet: Maximum Likelihood (MLE) & Maximum A Posteriori (MAP)

---

## 1. Maximum Likelihood Estimation (MLE)

**Simple explanation**
You have some data, and you assume it was generated by a distribution with an unknown parameter (e.g., a coin with unknown bias θ). MLE asks: *"Which value of the parameter would have made the data I actually observed most probable?"* You pick that value.

**Formula (general)**

$$
\theta^{*} = \arg\max_{\theta} \; p(\text{data} \mid \theta) = \arg\max_{\theta} \prod_{i=1}^{n} p(y^{(i)} \mid \theta)
$$

Because products of many small probabilities are numerically painful and hard to differentiate, we take the log (log is monotonic, so the maximizer doesn't change):

$$
\theta^{*} = \arg\max_{\theta} \sum_{i=1}^{n} \log p(y^{(i)} \mid \theta)
$$

**Worked example (coin flips)**
10 throws, 3 heads, 7 tails → maximize $\theta^3(1-\theta)^7$ → take derivative of the log-likelihood, set to 0 → $\theta^{*} = 3/10$ (exactly the observed frequency).

**Worked example (linear regression)**
If $y = x^\top w + \varepsilon$ with $\varepsilon \sim N(0,\sigma^2)$, then maximizing the Gaussian log-likelihood over $w$ is **mathematically identical** to minimizing the sum of squared errors:

$$
\arg\min_{w} \sum_{i=1}^n \left(y^{(i)} - (x^{(i)})^\top w\right)^2
$$

This is why "least squares regression" and "MLE under Gaussian noise" are the same thing.

**Pros**
- Simple, intuitive, and mathematically well-grounded.
- Makes no assumptions beyond the model itself (no need to specify prior beliefs).
- Directly connects to familiar loss functions (squared loss, cross-entropy, etc.) via the choice of noise/distribution.
- Asymptotically efficient/consistent — with enough data, it converges to the true parameter.

**Cons**
- Can overfit or give extreme/unrealistic estimates with little data (e.g., 0 heads out of 2 tosses gives θ* = 0, implying a coin that can *never* land heads — too strong a conclusion from 2 flips).
- Ignores any prior knowledge you might have about plausible parameter values.
- Sensitive to noise/outliers when data is scarce.

**When to use it**
- You have plenty of data.
- You have no meaningful prior belief about the parameters, or you deliberately want a "purely data-driven" estimate.

## 2. Maximum A Posteriori (MAP)

**Simple explanation**
MAP says: "Instead of only asking what parameter makes the data most likely, let's also factor in what we already believed about the parameter *before* seeing the data." You combine your **prior belief** with the **evidence from data** to get the most probable parameter *given* the data.

**Formula (via Bayes' rule)**

$$
p(\theta \mid \text{data}) = \frac{p(\text{data}\mid \theta)\cdot p(\theta)}{p(\text{data})}
$$

Since $p(\text{data})$ doesn't depend on θ, we only need:

$$
\theta^{*} = \arg\max_{\theta}\; p(\text{data}\mid\theta)\cdot p(\theta)
$$

**Worked example (coin flips with Beta prior)**
If $\theta \sim \text{Beta}(\alpha,\beta)$ (a natural prior for probabilities, bounded between 0 and 1):

$$
\theta^{*} = \frac{n_{heads} + \alpha - 1}{n_{heads}+\alpha-1 + n_{tails}+\beta-1}
$$

Intuition: it's as if you had already seen $\alpha - 1$ extra heads and $\beta - 1$ extra tails *before* your real experiment. With a strong prior (e.g., Beta(11,11), meaning "I expect a fair coin"), even 0 heads out of 2 tosses gives a sensible θ* ≈ 0.45 instead of the MLE's implausible θ* = 0.

**Worked example (linear regression with Gaussian prior on w)**
If $w \sim N(0, \tau^2 I)$, MAP becomes:

$$
\arg\min_{w} \sum_{i=1}^n \left(y^{(i)} - (x^{(i)})^\top w\right)^2 + \lambda \lVert w \rVert_2^2
$$

This is **ridge regression** (L2-regularized least squares), where $\lambda = \sigma^2/\tau^2$.

**Pros**
- More robust with small datasets — the prior "softens" extreme conclusions from limited evidence.
- Naturally leads to regularization, which helps prevent overfitting.
- Lets you inject domain knowledge into the model (e.g., "weights should be small," "a coin is probably close to fair").

**Cons**
- Requires choosing a prior — if the prior is wrong or too strong, it can bias your estimate away from the truth.
- More complex to set up (need to pick and justify a prior distribution and its hyperparameters).
- With very large amounts of data, the prior's influence becomes negligible, so the extra complexity may not be worth it.

**When to use it**
- You have limited data.
- You have genuine prior knowledge about the parameters (e.g., "weights shouldn't be huge," "coins are usually close to fair").
- You want built-in regularization to control overfitting.

## 3. The Big Equivalence: Risk Minimization ↔ Likelihood

This is the "aha" connection tying probability theory to the loss functions used in everyday ML.

| Empirical / Regularized Risk Minimization | Probabilistic View |
|---|---|
| Minimize | Maximize |
| Sum (of losses) | Product (of probabilities) → log turns it into a sum |
| Loss function | Noise distribution |
| L2-loss (squared error) | Gaussian noise |
| L1-loss (absolute error) | Laplacian noise |
| Regularizer (e.g., $\lambda\lVert w\rVert_2^2$) | Prior on parameters |
| Regularization strength λ | Ratio of variances $\sigma^2/\tau^2$ |
| **Unregularized risk minimization** | **= MLE** |
| **Regularized risk minimization** | **= MAP** |

**Simple explanation:** Choosing a loss function is secretly the same as assuming a noise distribution. Choosing a regularizer is secretly the same as assuming a prior on the parameters. MLE and "plain" loss minimization are the same idea in two different languages; MAP and regularized loss minimization are the same idea in two different languages.

## 4. MLE vs. MAP — Which One to Use?

| | MLE | MAP |
|---|---|---|
| Uses prior knowledge? | No | Yes |
| Treats parameter as random variable? | No (frequentist) | Yes (Bayesian) |
| Equivalent to | Empirical risk minimization | Regularized risk minimization |
| Best with | Lots of data | Little data / need for regularization |
| Risk | Overfitting / extreme estimates on small data | Bias if prior is wrong |

Neither is universally "more correct" — they answer slightly different questions (probability of data given parameter vs. probability of parameter given data), and the right choice depends on how much data you have and whether you have meaningful prior knowledge.

## 5. Exam Focus — Things to Watch Out For

- **Know the derivation cold**, not just the final formula: write out the likelihood → take the log → differentiate → set to 0 → solve. Professors love asking you to redo this for a new distribution (e.g., Poisson, exponential) or a new prior.
- **Don't forget "log doesn't change the argmax."** This is the standard trick that turns a product into a sum and makes differentiation tractable — be ready to justify *why* it's valid (log is a strictly monotonically increasing function).
- **Be able to go both directions**: given a loss function, name the implied noise distribution (and vice versa). Given a regularizer, name the implied prior (and vice versa). L2 loss ↔ Gaussian noise; L1 loss ↔ Laplace noise; L2 regularizer ↔ Gaussian prior (ridge); L1 regularizer ↔ Laplace prior (LASSO, often asked as an extension).
- **Know why MLE fails with small samples** — be ready to give the classic example (0 heads in 2 tosses → θ* = 0) and explain *why* this is a bad estimate.
- **Understand the "pseudo-count" interpretation of MAP with a Beta prior**: the parameters α, β act like "imaginary" extra observations added to your real data. This shows up often as a conceptual question.
- **Conjugate priors**: know that Beta is the conjugate prior for the Binomial/Bernoulli likelihood, and Gaussian is (self-)conjugate for a Gaussian likelihood with known variance. Conjugacy means the posterior has the same distributional family as the prior — this is why the math stays clean.
- **λ is not arbitrary** — be able to derive that $\lambda = \sigma^2/\tau^2$ in the Gaussian-noise/Gaussian-prior regression case. This connects regularization strength directly to how confident you are in your prior (small τ² = strong prior belief = large λ = more regularization).
- **MAP is not "fully Bayesian."** MAP only returns a single point estimate (the peak of the posterior), not the full posterior distribution. If asked to distinguish MAP from full Bayesian inference, mention that true Bayesian inference would integrate over the whole posterior rather than just taking its maximum.
- **Watch your signs and constants** when converting between "maximize log-likelihood" and "minimize negative log-likelihood/loss" — a very common source of small errors in exams.
- **i.i.d. assumption**: be ready to explain what independent and identically distributed means and why it's needed to turn a joint probability into a simple product.
